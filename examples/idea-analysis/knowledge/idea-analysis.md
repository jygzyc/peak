# Idea analysis knowledge

Deep analysis separates claims into four classes: observed problem, proposed
value, feasibility assumption, and delivery/adoption risk. A recommendation is
valid only when it cites evidence from each relevant class and states a cheap
experiment that could falsify the riskiest assumption.

For offline products, explicitly consider local persistence, conflict identity,
sync retry/idempotency, media size, authentication expiry, device loss, privacy,
auditability and operational recovery. Do not treat a feature list as evidence
of desirability or feasibility.
