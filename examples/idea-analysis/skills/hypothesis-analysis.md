# Skill: hypothesis-driven idea analysis

1. Extract the user, current behavior, pain and measurable target.
2. Identify the smallest value hypothesis that could change that behavior.
3. Test technical feasibility against explicit constraints, not an idealized
   architecture.
4. Rank risks by impact and uncertainty.
5. Synthesize a bounded recommendation from verified parent Facts.
6. Propose the cheapest experiment and a falsifiable success/failure threshold.
